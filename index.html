DNA Sequencing

Gene-Phenotype Associations via Variant Analysis

DNA sequencing lets you find differences between your sample and a known reference genome. These differences, variants and mutations, can be mapped to genes and those genes to biological traits and phenotypes.

1
DNA Extraction / Dataset Download
CTAB protocol, extraction kits, NCBI SRA, FASTQ files
+
What is lab extraction?
DNA is extracted from fresh biological tissue using the CTAB method, a chemical protocol that breaks open cells, removes proteins and RNA, and leaves behind purified genomic DNA. Extraction kits such as DNeasy from Qiagen are also widely used. The purified DNA is sent for whole-genome sequencing on Illumina machines at typically 30 to 40x coverage, meaning every genomic position is read approximately 40 times on average.
What are public databases?
NCBI SRA (Sequence Read Archive) holds thousands of publicly available sequencing datasets. Every project has SRA accession numbers such as SRR12345678. Visit NCBI SRA to search and find datasets. Download using SRA Toolkit and convert to FASTQ format.
What is a FASTQ file?
A text file containing millions of short sequence reads. Each read has four lines: a header ID, the DNA sequence in A, T, G, and C, a separator line, and a quality score line showing how confident the sequencing machine was at each base position.
CTAB extraction DNeasy Kit (Qiagen) SRA Toolkit v3.4.1 Input: tissue sample or SRA accession number Output: sample_1.fastq and sample_2.fastq 
# Download from NCBI SRA
prefetch SRR_ACCESSION
fasterq-dump SRR_ACCESSION.sra --split-files
Flag / argumentWhat it means
SRR_ACCESSIONThe unique ID of the dataset from NCBI SRA, for example SRR12345678
--split-filesSeparates output into two files: _1.fastq (forward reads) and _2.fastq (reverse reads)
2
Quality Check
FastQC, FastP, HTML quality report
+
What is quality checking?
Before any analysis, you assess whether your raw data is reliable. FastQC generates a detailed HTML report checking per-base quality scores, adapter contamination, GC content, and duplication levels. You want per-base quality above Q30, meaning 99.9% accuracy at every base.
What is a quality score?
Phred quality scores show how confident the sequencing machine is at each base. Q20 means 99% accuracy, Q30 means 99.9%, Q40 means 99.99%. For variant calling, Q30 is the recommended minimum.
FastQC FastP Input: .fastq files Output: HTML quality report Q30 above 80% and low adapter content: proceed. Otherwise trim first. 
fastqc sample_1.fastq sample_2.fastq \
  --outdir fastqc_results/ --threads 6
Flag / argumentWhat it means
sample_1.fastqThe FASTQ file to assess. List multiple files to process them all at once
--outdirThe folder where FastQC saves the HTML report and zip files
--threads 6Number of parallel CPU threads to use. More threads means faster processing
3
Trimming
Trimmomatic, BBDuk2, Trim Galore, then FastQC again
+
Why do we trim?
Raw reads contain adapter sequences from the sequencing kit accidentally attached to read ends, and low-quality bases at read ends where sequencing chemistry degrades. Both must be removed before alignment.
What is Trimmomatic and what do its parameters mean?
Trimmomatic is the most widely used trimming tool for Illumina paired-end data. ILLUMINACLIP removes adapter sequences by comparing reads against a reference adapter file. HEADCROP removes the first 10 bases from every read to eliminate hexamer priming bias from library preparation. LEADING removes low-quality bases from the start. TRAILING removes low-quality bases from the end. SLIDINGWINDOW scans in a window of bases and cuts when average quality drops below the threshold. MINLEN discards reads shorter than the minimum length after trimming.
What happens after trimming?
Always re-run FastQC on the trimmed files to confirm adapter content is gone and base quality has improved. This verification step must always happen before alignment.
Trimmomatic BBDuk2 Trim Galore Input: raw .fastq files Output: clean_1.fastq and clean_2.fastq 
trimmomatic PE sample_1.fastq sample_2.fastq \
  clean_1.fastq unpaired_1.fastq \
  clean_2.fastq unpaired_2.fastq \
  ILLUMINACLIP:TruSeq3-PE.fa:2:30:10 \
  HEADCROP:10 LEADING:3 TRAILING:3 \
  SLIDINGWINDOW:4:20 MINLEN:50

fastqc clean_1.fastq clean_2.fastq
Flag / argumentWhat it means
PEPaired-end mode. Processes both forward and reverse read files together
ILLUMINACLIP:TruSeq3-PE.fa:2:30:10Remove Illumina TruSeq3 adapters. Numbers control seed mismatches, palindrome clip threshold, and simple clip threshold
HEADCROP:10Remove the first 10 bases from every read to eliminate hexamer priming bias
LEADING:3Remove bases from the start of a read if quality is below 3
TRAILING:3Remove bases from the end of a read if quality is below 3
SLIDINGWINDOW:4:20Scan with a 4-base window and cut when average quality in that window drops below 20
MINLEN:50Discard reads shorter than 50 bases after all trimming
4
Alignment to Reference Genome
BWA, Bowtie2, SAM output
+
What is alignment?
Alignment is the process of taking each short read and finding its exact position on the reference genome. Think of it as placing millions of tiny puzzle pieces onto a completed picture.
What is a reference genome?
A high-quality sequence assembled from a representative group of healthy individuals of that species. It serves as the standard map your sample is compared against. For rice the reference is IRGSP-1.0 (Nipponbare). For humans it is GRCh38. For Arabidopsis it is TAIR10.1.
What is BWA?
BWA (Burrows-Wheeler Aligner) uses the Burrows-Wheeler Transform algorithm to compress and index the genome for ultrafast searching. The BWA-MEM mode is the gold standard for DNA short-read alignment worldwide.
What is Bowtie2?
Bowtie2 is another fast, memory-efficient short-read aligner using an algorithm called the FM-index. Well suited for DNA alignment and commonly used in ChIP-seq and ATAC-seq applications.
What is a SAM file?
SAM (Sequence Alignment Map) is a large text file where each line represents one read, recording which chromosome it mapped to, which position, which strand, the mapping quality score (MAPQ), and any mismatches to the reference.
BWA v0.7.17 Bowtie2 Input: clean .fastq files and reference.fa Output: aligned.sam 
# Index the reference genome once before aligning
bwa index genome.fa

# Align paired-end reads
bwa mem -t 8 genome.fa \
  clean_1.fastq clean_2.fastq \
  > aligned.sam
Flag / argumentWhat it means
bwa indexBuilds an index of the reference genome. Only needs to be done once per reference
bwa memThe BWA-MEM alignment algorithm, recommended for reads of 70bp and longer
-t 8Number of threads to use. The -t flag means threads. Set this to match your available CPU cores
genome.faThe reference genome FASTA file
clean_1.fastq clean_2.fastqThe two trimmed input files: forward reads and reverse reads
> aligned.samThe greater-than symbol redirects the output into a file called aligned.sam instead of printing to the screen

SAM to BAM, Sorting, Marking Duplicates, Indexing, Validation
SAMtools, Picard Tools
+
What is a BAM file?
BAM is the binary compressed version of a SAM file. Same information, far smaller and faster for tools to read. All downstream tools require BAM rather than SAM.
Why do we sort?
Downstream tools including variant callers and IGV require reads ordered by chromosomal position, not the order they were sequenced in.
Why do we mark and remove duplicates?
PCR amplification before sequencing creates duplicate copies of some fragments. These duplicates inflate read depth and can make a sequencing error look like a real variant. Picard MarkDuplicates flags them and SAMtools rmdup removes them.
Why do we index?
Indexing creates a .bai file alongside the BAM. This acts like a table of contents, allowing tools to jump directly to any chromosomal position without reading the entire file.
Why do we validate?
SAMtools validate checks that the BAM file is not corrupted and conforms to the SAM/BAM specification before variant calling begins.
SAMtools Picard Tools Input: aligned.sam Output: deduped.bam and deduped.bam.bai
aligned.sam → convert to BAM → sort by coordinate → mark duplicates → remove duplicates → index → validate
# Convert SAM to BAM and sort
samtools view -bS aligned.sam | samtools sort -o sorted.bam

# Mark PCR duplicates
picard MarkDuplicates I=sorted.bam O=marked.bam M=metrics.txt

# Remove duplicates
samtools rmdup marked.bam deduped.bam

# Index the final BAM
samtools index deduped.bam

# Validate before proceeding
samtools validate deduped.bam
Flag / argumentWhat it means
samtools viewReads and converts SAM or BAM files
-bOutput in BAM format (binary compressed)
-SInput is in SAM format (text). Tells SAMtools what format it is reading
| samtools sortThe pipe character passes output directly into the sort command without creating an intermediate file
-o sorted.bamOutput file name for the sorted BAM. The -o flag means output file
I=sorted.bamPicard input file. I stands for input
O=marked.bamPicard output file. O stands for output
M=metrics.txtA file where Picard writes duplication statistics. M stands for metrics
samtools rmdupRemoves reads flagged as duplicates by Picard

samtools validateChecks the BAM file for corruption or format errors

Variant Calling
FreeBayes, GATK HaplotypeCaller, VCFtools, VCF output
+
What is variant calling?
The variant caller reads your BAM file and compares your sample's nucleotides to the reference genome position by position. Wherever your sample consistently shows a different nucleotide, that position is called a variant.
What types of variants are there?
SNPs (single nucleotide polymorphisms) are single letter changes at one position, for example A becomes G. InDels are insertions or deletions of one or more bases. Structural variants are larger rearrangements of the genome.
What is FreeBayes?
FreeBayes is a Bayesian genetic variant detector. It calculates the probability that an observed difference is a real variant versus a sequencing error. It requires minimum depth and mapping quality thresholds before calling a variant.
What is GATK HaplotypeCaller?
The Broad Institute standard variant caller. It uses local de-novo assembly of haplotypes around each potential variant site for higher accuracy, especially in complex genomic regions.
What is VCFtools?
VCFtools filters, summarises, and manipulates VCF files after variant calling. It applies depth, quality, and genotype filters to remove unreliable variant calls.
What is a VCF file?
VCF (Variant Call Format) is a text file listing every variant position found. Each line contains the chromosome, position, reference allele, alternate allele, quality score, filter status, and detailed read support information.
FreeBayes GATK HaplotypeCaller VCFtools Input: deduped.bam and reference.fa Output: variants.vcf then filtered.vcf 
freebayes -f genome.fa deduped.bam > variants.vcf

vcftools --vcf variants.vcf \
  --minDP 10 --minQ 40 \
  --recode --out filtered
Flag / argumentWhat it means
-f genome.faThe reference genome FASTA file. The -f flag means reference FASTA file
deduped.bamThe de-duplicated BAM file to call variants from
> variants.vcfRedirects the output into a VCF file
--vcf variants.vcfTells VCFtools which VCF file to filter
--minDP 10Minimum depth: only keep variants supported by at least 10 reads
--minQ 40Minimum quality score: only keep variants with quality above 40
--recodeWrite out a new VCF file with only the passing variants
--out filteredPrefix name for the output file. Output will be called filtered.recode.vcf

IGV Visualisation and Variant Analysis
Integrative Genomics Viewer, visual confirmation of variants and mutations
+
What is IGV?
IGV (Integrative Genomics Viewer) is a desktop tool that lets you visually inspect aligned reads at any chromosomal position. You load your BAM file and VCF file, then zoom into any position to see all reads piled up at that site.
What is variant analysis in IGV?
At a true SNP position you will see a column of coloured mismatching bases, for example all reads show G where the reference shows A. For an InDel you will see gaps or insertions clearly in the read stack. This visual check confirms whether a called variant is real or an artefact.
What is a mutant check?
In a mutant study, you compare your sample to the reference to see what has changed. IGV shows you exactly which nucleotide changed, at which chromosomal coordinate, and how many reads support it. This validates your VCF calls before annotation.
IGV (desktop application) Input: deduped.bam and filtered.vcf and reference.fa Output: visual confirmation of SNP and InDel type and location
Load .bam and .vcf in IGV, navigate to variant position, view read pile-up
SNP: column of mismatched bases. InDel: gaps or insertions in reads.

Variant Annotation
VEP (Ensembl), SnpEff, functional consequence, SIFT score
+
What is variant annotation?
Your VCF file contains chromosomal positions and nucleotide changes but no biological context. Annotation adds that context by looking up each variant position in gene databases and telling you what gene is there and what the change means biologically.
What does VEP tell you?
VEP (Variant Effect Predictor) from Ensembl tells you which gene sits at that position, which transcript is affected, and what the nucleotide change does to the protein. Missense means the amino acid changes. Synonymous means no protein change. Stop-gained means the protein is truncated. Frameshift means the reading frame is disrupted. SIFT scores predict whether the amino acid change is tolerated or likely to disrupt protein function.
VEP (Ensembl) SnpEff SIFT score Input: filtered.vcf Output: annotated VCF with gene names and variant consequences
Ensembl VEP SnpEff

SNP Validation — QTL-seq and KASP Assays
QTL-seq, KASP assays, TASSEL, association mapping
+
What is QTL-seq?
QTL-seq is a bulk segregant approach to map quantitative traits to chromosomal regions. You pool DNA from individuals at opposite extremes of a phenotypic distribution into two bulks and sequence both. Where the bulks consistently differ in SNP patterns, that region controls the trait. The output is a delta-SNP-index plot showing QTL peaks on chromosomes.
What are KASP assays?
KASP (Kompetitive Allele-Specific PCR) validates specific SNPs from the QTL region across large populations. Each KASP assay targets one SNP and distinguishes the two alleles by fluorescence. This is SNP validation, confirming that computationally identified SNPs are real and biologically associated with the phenotype.
What is TASSEL?
TASSEL performs statistical association mapping between SNP genotypes and phenotypic data. Download TASSEL software at maizegenetics.net/tassel. It uses a mixed linear model (MLM) that accounts for population structure and kinship between individuals.
QTL-seq KASP assays TASSEL Input: pooled bulk BAM files or individual genotype data Output: delta-SNP-index plot, QTL peaks, validated SNP markers
TASSEL software

Pathway Enrichment and Gene Ontology
Gene Ontology, KEGG, STRING, AgriGO, Panther DB
+
What is pathway analysis?
After annotation you have a list of genes affected by variants. Pathway analysis asks whether these genes cluster in specific biological processes and whether that clustering makes sense for your trait.
What is Gene Ontology?
Gene Ontology classifies genes into biological process, molecular function, and cellular component. Enrichment analysis tells you which GO terms appear more often in your gene list than expected by chance. Visit geneontology.org.
What is KEGG?
KEGG maps your genes to known metabolic and signalling pathways with visual pathway diagrams. Visit kegg.jp.
What is STRING?
STRING shows protein-protein interaction networks among your genes. Genes that interact physically often function together. Visit string-db.org.
What is Panther DB?
Panther DB provides evolutionary and functional classification of genes across species. Visit pantherdb.org.
Input: annotated gene list Output: enriched pathways and trait associations
Gene Ontology KEGG STRING AgriGO v2 Panther DB

RNA Sequencing

RNA Sequencing Analysis

RNA reflects what a cell is actively doing at any given moment. RNA-seq captures gene activity across conditions, time points, and at the resolution of how individual genes are spliced. Choose your analysis pathway below.

1
RNA Extraction / Dataset Download
RNeasy kit, TRIzol, NCBI SRA, FASTQ files
+
What is RNA extraction?
RNA is extracted from fresh tissue using kits such as RNeasy from Qiagen or TRIzol. RNA degrades very rapidly so samples must be snap-frozen in liquid nitrogen immediately and stored at minus 80 degrees Celsius. RNA integrity is assessed using a BioAnalyzer (RIN score above 7 required) and quantified with Qubit. Library preparation converts RNA to cDNA with sequencing adapters, yielding approximately 100 million paired-end reads per sample.
What about public data?
RNA-seq datasets are publicly available on NCBI SRA. Downloaded and converted to FASTQ using SRA Toolkit.
RNeasy Kit (Qiagen) TRIzol SRA Toolkit Input: tissue sample or SRA accession Output: sample_1.fastq and sample_2.fastq

Alignment — STAR / HISAT2
Splice-aware aligners, BAM output, IGV strandedness check
+
Why do RNA-seq alignments need splice-aware tools?
RNA-seq reads come from mature mRNA with introns removed. A single read can span two exons thousands of bases apart on the genome. Standard DNA aligners cannot handle this. Splice-aware aligners understand that reads can jump across intron gaps.
What is STAR?
STAR (Spliced Transcripts Alignment to a Reference) is the most widely used RNA-seq aligner. It builds a genome index and aligns reads in two passes, first discovering splice junctions then aligning using those junctions. Very fast and handles novel splice sites.
What is HISAT2?
HISAT2 is a graph-based splice-aware aligner using less memory than STAR. Good alternative on memory-limited systems. Analysis using RStudio for R-based steps.
Why check strandedness in IGV?
After alignment, loading the BAM in IGV lets you visually verify alignment quality and determine whether the library is stranded or unstranded. This affects parameters in downstream tools.
STAR HISAT2 IGV strandedness check RStudio (for R steps) Input: clean .fastq files, reference genome, GTF annotation Output: sorted .BAM file 
STAR --runThreadN 8 \
  --genomeDir ./star_index/ \
  --readFilesIn clean_1.fastq clean_2.fastq \
  --outSAMtype BAM SortedByCoordinate \
  --outSAMattributes NH HI AS NM \
  --outFileNamePrefix ./output/sample_
samtools index ./output/sample_.Aligned.sortedByCoord.out.bam
Flag / argumentWhat it means

Feature Counts — Count Matrix Generation
featureCounts, HTSeq, Gene x Sample count matrix
+
What is a count matrix?
After alignment, you count how many reads landed on each gene in each sample. The result is a matrix with genes as rows and samples as columns. Each cell is the number of reads from that sample mapping to that gene. More reads means higher gene expression.

  -o counts_matrix.txt \
  sample1.bam sample2.bam sample3.bam

Differential Gene Expression — DESeq2 / edgeR
R, RStudio, statistical testing, volcano plots, expression tables
+
What is DESeq2?
DESeq2 is an R package run in RStudio. It determines which genes are significantly differentially expressed between conditions. It normalises for library size differences, applies a negative binomial statistical model, and uses shrinkage estimation for accurate fold-change values even with small sample sizes.
What is the significance threshold?
A gene is differentially expressed if adjusted p-value (FDR) is below 0.05 and absolute log2 fold change is above or equal to 1 (a 2x difference or more). Upregulated means more active in condition A. Downregulated means less active.
How are results visualised?
Volcano plots show all genes as dots where x-axis is fold change and y-axis is significance. Upper right: significantly upregulated. Upper left: significantly downregulated. Heatmaps and PCA plots also explore expression patterns.
DESeq2 (R) edgeR (R) EnhancedVolcano (R) RStudio Input: count matrix Output: DGE results table and volcano plots Significant: padj below 0.05 and absolute log2FC above or equal to 1 
library(DESeq2)
dds <- DESeqDataSetFromMatrix(
  countData=counts, colData=coldata,
  design=~condition)
dds <- DESeq(dds)
res <- results(dds)
write.csv(as.data.frame(res),"DGE_results.csv")
Flag / argumentWhat it means
countData=countsThe raw count matrix from featureCounts
colData=coldataA table describing each sample including which condition or group it belongs to
design=~conditionThe statistical model formula. The tilde (~) means modelled by. Tells DESeq2 to test differences based on the condition column
DESeq(dds)Runs the full DESeq2 analysis: normalisation, dispersion estimation, and statistical testing
results(dds)Extracts the results table with log2 fold change, p-value, and adjusted p-value for every gene
write.csvSaves the results table as a CSV file openable in Excel

SpliSER Process
SpliSER v1.0.4, alpha and beta read counting per splice site
+
What is SpliSER?
SpliSER counts two types of reads at every splice site. Alpha reads cross the splice junction, spanning the exon-exon boundary where the intron was removed. Beta reads end at the splice site without crossing. The ratio gives a Splice Site Efficiency (SSE) score between 0 and 1. SpliSER process is run independently on each BAM file.
SpliSER v1.0.4 Input: sample.bam and annotation.gtf Output: per-sample splice site counts (.tsv file) 
spliser process sample.bam \
  -o spliser_output_sample \
  -A annotation.gtf
# For stranded data add -s RF or -s FR

processSpliSER subcommand that counts alpha and beta reads per splice site for one BAM file

-A annotation.gtfThe GTF annotation file defining gene and splice site positions
-s RF or -s FRStrandedness flag. RF means reverse-forward stranded library. FR means forward-reverse. Leave out for unstranded data

SpliSER Combine and Output
Merges all sample outputs, SSE normalisation, diffSpliSER format
+
What do SpliSER combine and output do?
SpliSER combine merges all per-sample outputs into one unified dataset, interpolating missing counts for splice sites absent in any sample. SpliSER output then converts raw counts into normalised SSE values and formats the data for diffSpliSER statistical testing using the -t flag.

  -o spliser_combined_all_samples

spliser output -C spliser_combined_all_samples \
  -o final_spliser_output.tsv \
  -t diffspliSER

diffSpliSER — Pairwise Statistical Testing
edgeR GLM framework, FDR below 0.05, absolute dSSE above or equal to 0.1
+
What is diffSpliSER?
diffSpliSER uses the edgeR generalised linear model framework to statistically test whether splice site efficiency differs between every pair of conditions. A splice site is significant if FDR is below 0.05 and the difference in SSE between groups is at least 0.1. This ensures only biologically meaningful splicing changes are reported. The full diffSpliSER R Markdown pipeline is available on GitHub.
edgeR (R) RStudio Input: SSE matrix and target file with sample groups and library sizes Output: pairwise results files (_hits.tsv)
diffSpliSER R Markdown (GitHub)

Venn Diagram and PCA Analysis
VennDiagram, ggplot2 in RStudio, reproducibility across comparisons
+
Why run Venn diagrams and PCA?
Venn diagrams compare results across multiple pairwise comparisons to find splice sites reproducibly differentially spliced across all comparisons. The overlap represents the most reliable and biologically meaningful sites. PCA on SSE values confirms whether samples cluster as expected by condition or time point, validating experimental design and data quality.
VennDiagram (R) ggplot2 and prcomp (R) RStudio Output: Venn diagrams and PCA plots

Motif Enrichment Analysis — XSTREME
MEME Suite, bedtools getfasta, RBP motif database
+
What is motif enrichment analysis?
A custom-sized window of bases centred on each differentially spliced site is extracted from the reference genome using bedtools getfasta. These sequences are searched against a database of known RNA-binding protein motifs to identify which RNA-binding proteins may be regulating splicing at those sites. Control sequences are randomly chosen sites from the reference genome. The number of control sequences (2000 to 10000) determines how stringent the background comparison is. Run via the MEME Suite web server.
XSTREME (MEME Suite) bedtools getfasta RBP motif database Input: splice site coordinates converted to FASTA sequences Output: enriched RNA-binding protein motifs at differentially spliced sites
MEME Suite XSTREME 
bedtools getfasta -fi reference.fa \
  -bed splice_sites.bed \
  -fo test_sequences.fa -s -name
Flag / argumentWhat it means
-fi reference.faThe reference genome FASTA file to extract sequences from
-bed splice_sites.bedA BED file listing chromosomal coordinates of each splice site window
-fo test_sequences.faThe output FASTA file containing the extracted sequences
-sStrand-aware extraction. Negative strand sequences are automatically reverse complemented
-nameUses the name field from the BED file as the FASTA sequence header

Alignment — TopHat
TopHat, splice-aware, uses Bowtie2 internally, BAM output
+
What is TopHat?
TopHat is a splice-aware RNA-seq aligner that uses Bowtie2 as its underlying alignment engine. It first aligns reads that map continuously, then handles reads spanning splice junctions by discovering exon-exon boundaries from the data. Produces sorted BAM files ready for Cufflinks. Now considered legacy but widely cited in scientific literature.
TopHat Bowtie2 (used internally) Input: clean .fastq files and reference genome Output: accepted_hits.bam and unmapped.bam 
# Build Bowtie2 index once
bowtie2-build genome.fa genome_index

# Align with TopHat
tophat -p 8 \
  -G annotation.gtf \
  -o tophat_output/ \
  genome_index \
  clean_1.fastq clean_2.fastq
Flag / argumentWhat it means
bowtie2-buildBuilds a Bowtie2 index of the reference genome. TopHat uses this internally so it must be built first
-p 8Number of parallel processing threads. The -p flag means processors
-G annotation.gtfProvides known gene annotation to guide splice junction detection. The -G flag means GTF annotation
-o tophat_output/Output directory for all TopHat result files. The -o flag means output directory
genome_indexThe Bowtie2 index prefix to align against

Cufflinks — Transcript Assembly
Cufflinks, assembles transcripts, estimates expression as FPKM
+
What is Cufflinks?
Cufflinks takes aligned BAM files from TopHat and assembles them into transcripts, then estimates expression levels. Unlike featureCounts which counts reads per known gene, Cufflinks can discover novel transcripts and quantify at the isoform level. Expression is reported as FPKM (Fragments Per Kilobase of transcript per Million mapped reads), which normalises for both sequencing depth and transcript length.
Cufflinks Input: accepted_hits.bam and annotation.gtf Output: transcripts.gtf and FPKM expression values 
cufflinks -p 8 \
  -G annotation.gtf \
  -o cufflinks_output/ \
  tophat_output/accepted_hits.bam
Flag / argumentWhat it means
-p 8Number of parallel processing threads
-G annotation.gtfReference annotation. Cufflinks uses this to guide assembly and report known transcript expression. The -G flag means reference GTF
-o cufflinks_output/Output directory for this sample's Cufflinks results
accepted_hits.bamThe TopHat aligned BAM file for this sample

Cuffmerge — Merge Transcript Assemblies
Cuffmerge, unified annotation across all samples
+
What is Cuffmerge?
When Cufflinks is run on multiple samples, each produces its own transcript assembly. Cuffmerge merges all of these into a single unified annotation, reconciling overlapping and novel transcripts. This merged annotation is then used as input for Cuffdiff to ensure all samples are compared on the same set of transcripts.
Cuffmerge Input: list of transcripts.gtf files from all samples Output: merged.gtf (unified transcript annotation) 
ls cufflinks_*/transcripts.gtf > assemblies.txt

cuffmerge -g annotation.gtf \
  -s genome.fa \
  -o cuffmerge_output/ \
  assemblies.txt
Flag / argumentWhat it means
ls ... > assemblies.txtLists all Cufflinks transcript files and saves them into a text file that Cuffmerge reads
-g annotation.gtfReference annotation used to compare against known transcripts during merging
-s genome.faReference genome sequence used to determine strand of novel transcripts
-o cuffmerge_output/Output directory for the merged annotation
assemblies.txtThe text file where each line is the path to one sample's transcripts.gtf

Cuffdiff — Differential Expression
Cuffdiff, gene and transcript level, differential splicing, promoter use
+
What is Cuffdiff?
Cuffdiff finds genes and transcripts differentially expressed between conditions. Unlike DESeq2 which works on gene-level counts, Cuffdiff works at the isoform level. It detects which specific transcript isoforms changed, differential splicing (changes in isoform ratios), differential promoter usage, and differential coding output changes.
Cuffdiff Input: merged.gtf and BAM files for each condition Output: gene_exp.diff and isoform_exp.diff and splicing.diff 
cuffdiff -p 8 \
  -o cuffdiff_output/ \
  cuffmerge_output/merged.gtf \
  control_1.bam,control_2.bam \
  treated_1.bam,treated_2.bam
Flag / argumentWhat it means
-p 8Number of parallel processing threads
-o cuffdiff_output/Output directory for all Cuffdiff result files
merged.gtfThe unified transcript annotation from Cuffmerge
control_1.bam,control_2.bamBAM files for the control group separated by commas with no spaces
treated_1.bam,treated_2.bamBAM files for the treated group. Cuffdiff compares these against the control group

CummeRbund — Visualisation
CummeRbund R package in RStudio, abundance plots, differential expression plots
+
What is CummeRbund?
CummeRbund is an R package run in RStudio specifically designed to visualise and interpret Cuffdiff output. It reads the Cuffdiff results database and provides functions for producing scatter plots, volcano plots, heatmaps, boxplots of individual gene expression, and PCA plots.
CummeRbund (R) RStudio Input: Cuffdiff output directory Output: expression plots and differentially expressed gene lists 
library(cummeRbund)
cuff <- readCufflinks("cuffdiff_output/")
gene_diff <- diffData(genes(cuff))
csVolcanoMatrix(genes(cuff))
