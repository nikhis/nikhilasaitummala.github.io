DNA Sequencing

Gene-Phenotype Associations via Variant Analysis

DNA sequencing lets you find differences between your sample and a known reference genome. These differences — variants and mutations — can be mapped to genes, and those genes to biological traits and phenotypes. Each step below is clickable for full detail.

1
DNA Extraction / Dataset Download
CTAB protocol · Illumina WGS · NCBI SRA · FASTQ files
+
What is this step? Before any analysis, you need your raw DNA data. You either extract DNA fresh from biological tissue in the lab, or download publicly available sequencing data from NCBI's SRA (Sequence Read Archive).

Lab extraction: DNA is extracted using the CTAB method — a chemical protocol that breaks open cells, removes proteins and RNA, and leaves you with purified genomic DNA. This is then sent for whole-genome sequencing on Illumina machines (HiSeq, NovaSeq) at typically 30-40x coverage, meaning every genomic position is read ~40 times for high confidence.

Public databases: NCBI SRA holds thousands of publicly available sequencing datasets. Every project has SRA accession numbers (e.g. SRR12345678). You download them using SRA Toolkit and convert to FASTQ format.

What is a FASTQ file? A text file containing millions of short sequence reads. Each read has four lines: a header ID, the DNA sequence (A/T/G/C), a separator, and a quality score line showing how confident the sequencing machine was at each base position.
SRA Toolkit v3.4.1 prefetch + fasterq-dump CTAB extraction Input: tissue / SRA accession Output: .fastq files (paired-end: _1.fastq + _2.fastq) 
# Download from NCBI SRA
prefetch SRR12345678
fasterq-dump SRR12345678.sra --split-files
# Output: SRR12345678_1.fastq (forward reads)
#         SRR12345678_2.fastq (reverse reads)
2
Quality Check — FastQC
FastQC v0.11.2 · FastP · HTML quality report
+
What is quality checking? Before doing any analysis, you must assess whether your raw sequencing data is reliable. FastQC reads your FASTQ file and generates a detailed HTML report with visual plots.

What does FastQC tell you? Per-base sequence quality (you want Q30+, meaning 99.9% accuracy at every base), adapter contamination (lab kit sequences accidentally included in your reads), GC content distribution (should match your organism), sequence duplication levels, and N content (unknown bases).

What is a quality score? Phred quality scores indicate how confident the sequencing machine is. Q20 = 99% accuracy, Q30 = 99.9%, Q40 = 99.99%. For variant calling, you generally want Q20 minimum and aim for Q30+.
FastQC v0.11.2 FastP v0.23.2 Input: .fastq files Output: HTML quality report If Q30 > 80% and low adapter = proceed. Else trim first. 
fastqc sample_1.fastq sample_2.fastq \
  --outdir fastqc_results/ --threads 6
# Opens as HTML in browser — check green ticks
3
Trimming and Quality Control
Trimmomatic v0.40 · BBDuk2 · Trim Galore! · then FastQC again
+
Why trim? Raw sequencing reads contain two types of noise: adapter sequences (short synthetic sequences from the sequencing kit that accidentally get attached to the ends of your reads) and low-quality bases (positions where the sequencing machine was uncertain, usually at read ends).

What does Trimmomatic do? It removes adapter sequences using a reference adapter file, crops the first few bases (HEADCROP:10) to remove hexamer primer bias, trims low-quality bases from both ends (LEADING:3 TRAILING:3), applies a sliding window approach to cut reads where quality drops (SLIDINGWINDOW:4:20 means cut where any 4-base window drops below Q20), and discards reads shorter than 50 bases (MINLEN:50).

After trimming: Always re-run FastQC to confirm the trimming worked. Adapter content should be gone and base quality should improve.
Trimmomatic v0.40 BBDuk2 v37.36 (Q20 threshold) Trim Galore! v0.6.7 Input: raw .fastq files Output: clean_1.fastq + clean_2.fastq (paired) 
trimmomatic PE -threads 6 \
  sample_1.fastq sample_2.fastq \
  clean_1.fastq unpaired_1.fastq \
  clean_2.fastq unpaired_2.fastq \
  ILLUMINACLIP:TruSeq3-PE.fa:2:30:10 \
  HEADCROP:10 LEADING:3 TRAILING:3 \
  SLIDINGWINDOW:4:20 MINLEN:50

# Re-run FastQC to verify improvement
fastqc clean_1.fastq clean_2.fastq
4
Alignment to Reference Genome — BWA
BWA v0.7.17 (Burrows-Wheeler Aligner) · SAM output
+
What is alignment? Alignment is the process of taking each short read from your FASTQ file and finding its exact position on the reference genome — like placing puzzle pieces onto a completed picture.

What is a reference genome? A reference genome is a high-quality, well-annotated sequence assembled from a large group of healthy, typical individuals of that species. For rice: IRGSP-1.0 (Nipponbare). For humans: GRCh38. It represents the standard, and your sample is compared to it.

What is BWA? BWA (Burrows-Wheeler Aligner) uses an algorithm called the Burrows-Wheeler Transform — a way of compressing and indexing the genome so it can be searched extremely fast. BWA-MEM is the standard mode, optimised for reads of 70bp-1Mbp. It is the gold standard for DNA alignment.

What is a SAM file? The output. SAM (Sequence Alignment/Map) is a large text file where each line represents one read and records: which chromosome, which position, which strand, and how well it aligned (MAPQ score).
BWA v0.7.17 Algorithm: Burrows-Wheeler Transform + FM-index Input: clean_1.fastq + clean_2.fastq + reference.fa Output: aligned.sam
clean reads + reference genome.fa → BWA mem → aligned.sam
# Index reference genome (once)
bwa index genome.fa

# Align paired-end reads
bwa mem -t 8 genome.fa \
  clean_1.fastq clean_2.fastq \
  > aligned.sam
5
SAM to BAM — SAMtools Sorting, Indexing, Validation
SAMtools v1.3.1 · Picard Tools · SAM → BAM → sorted → deduplicated
+
What is a BAM file? BAM is the binary, compressed version of a SAM file. It is much smaller and faster to work with. The same information is stored, but in a format computers can read far more efficiently.

Why sort? Downstream tools (variant callers, IGV) require reads to be sorted by their chromosomal position, not the order they were sequenced in.

Why remove duplicates? During PCR amplification before sequencing, some DNA fragments get copied multiple times. These duplicates can artificially inflate read depth at a position and make a variant look more confident than it is. Picard MarkDuplicates flags and removes them.

Why index? Indexing creates a .bai file — like a table of contents. It allows tools to jump directly to any chromosomal position without reading the entire BAM file from the start.
SAMtools v1.3.1 Picard Tools Input: aligned.sam Output: deduped.bam + deduped.bam.bai (index)
aligned.sam → view (SAM to BAM) → sort → remove PCR duplicates → index → deduped.bam.bai
# Convert SAM to BAM and sort
samtools view -bS aligned.sam | \
  samtools sort -o sorted.bam

# Remove PCR duplicates
samtools rmdup sorted.bam deduped.bam

# Index for fast access
samtools index deduped.bam

# Validate the BAM file
samtools flagstat deduped.bam
6
Variant Calling — FreeBayes / GATK HaplotypeCaller
FreeBayes v1.1.0 · GATK HaplotypeCaller · Picard Tools · VCF output
+
What is variant calling? The variant caller reads your BAM file and, position by position across the entire genome, compares your sample's nucleotides to the reference. Wherever your sample consistently shows a different nucleotide, that's a variant.

Types of variants: SNPs (Single Nucleotide Polymorphisms) — a single letter change, e.g. A becomes G at a specific position. InDels — insertions or deletions of one or more bases. Structural variants — larger rearrangements.

What is FreeBayes? A Bayesian genetic variant detector. It looks at all the reads piled up at each position and calculates the probability that an observed difference is a real variant vs a sequencing error. It requires a minimum depth of 10 reads (DP > 10) and mapping quality > 40 before calling a variant.

What is GATK HaplotypeCaller? The industry-standard variant caller from the Broad Institute. It uses local de-novo assembly of haplotypes around each potential variant site for higher accuracy.

What is a VCF file? Variant Call Format. A text file listing every position where a variant was found, with columns for chromosome, position, reference allele, alternate allele, quality score, filter status, and detailed information about read support.
FreeBayes v1.1.0 GATK HaplotypeCaller Picard Tools Input: deduped.bam + reference.fa Output: variants.vcf → filtered.vcf Filter: depth >= 10, mapping quality >= 40 
# FreeBayes variant calling
freebayes -f genome.fa deduped.bam > variants.vcf

# Filter by depth and quality
vcffilter -f "DP > 10 & QUAL > 40" \
  variants.vcf > filtered.vcf
7
IGV Visualisation — Mutant Check
Integrative Genomics Viewer · Visual validation of variants
+
What is IGV? The Integrative Genomics Viewer is a desktop tool that lets you visually inspect your aligned reads at any chromosomal position. You load your BAM file and your VCF file, then zoom into any position to see the reads piled up like a mountain.

What do you look for? At a true SNP position, you will see a column of coloured mismatching bases — for example, all reads at position chr3:12345 show a G where the reference shows A. That is a clear SNP. For an InDel, you'll see gaps or insertions clearly marked in the read stack.

Mutant vs polymorphism: In a mutant study, the variant is a change from the wild-type. In a population study, it is a naturally occurring polymorphism. IGV helps you visually confirm which type you are seeing and whether it's in a coding region, intron, or intergenic region.
IGV (desktop) Input: deduped.bam + filtered.vcf + reference.fa Output: visual confirmation of variants
Load .bam + .vcf in IGV → navigate to variant position → read pile-up view
SNP = column of mismatched bases  |  InDel = gaps/insertions in reads
8
Variant Annotation — VEP / Ensembl
VEP · SnpEff · Ensembl database · functional consequence
+
What is annotation? Your VCF contains chromosomal positions and nucleotide changes — but no biological context. Annotation adds that context by looking up each position in gene databases.

What does VEP tell you? Which gene is at this position. Which transcript is affected. What the nucleotide change does to the protein: missense (amino acid changes), synonymous (same amino acid, silent), stop-gained (protein is truncated), frameshift (reading frame disrupted), 5' or 3' UTR (regulatory region affected). SIFT scores predict whether the amino acid change is tolerated or deleterious.

Why does this matter? A missense mutation in an active site of a kinase is far more likely to affect function than a synonymous change in an intron. Annotation lets you prioritise which variants are biologically interesting.
VEP (Ensembl) SnpEff SIFT score Input: filtered.vcf Output: annotated VCF with gene names + consequences
9
Pathway Enrichment — Gene Ontology / KEGG / STRING
Gene linkage to traits and biological pathways
+
What is pathway analysis? After annotation, you have a list of genes affected by variants. Pathway analysis asks: do these genes cluster in specific biological processes, and does that clustering make sense for your trait?

Gene Ontology (GO): Classifies genes by biological process (what pathway), molecular function (what activity), and cellular component (where in the cell). Enrichment analysis tells you which GO terms are over-represented in your gene list compared to what you'd expect by chance.

KEGG: Maps your genes to known metabolic and signalling pathways with visual pathway diagrams showing where your genes fit.

STRING: Shows protein-protein interaction networks among your genes. Genes that interact physically often function together — useful for understanding the mechanism behind a phenotype.
Gene Ontology KEGG STRING QTARO (rice QTL database) Input: annotated gene list Output: enriched pathways + trait associations
10
QTL-seq + KASP + SNP Validation
QTL-seq v2.2.3 · TASSEL 5.2.88 · KASP assays · SNP-seek
+
What is QTL-seq? A bulk segregant approach to map quantitative traits to chromosomal regions. You pool DNA from the most extreme individuals at each end of a phenotypic distribution (e.g. earliest vs latest heading plants), sequence both pools, and calculate the delta-SNP-index at every position. A peak in the delta-SNP-index indicates a QTL — a genomic region controlling the trait.

KASP assays: Once a QTL region is identified, KASP (Kompetitive Allele-Specific PCR) assays validate individual SNPs in that region across large plant populations. Each KASP assay targets one SNP and distinguishes the two alleles by fluorescence. We validated 49 SNPs in the qDTH3 region of rice chromosome 3.

TASSEL: Statistical association mapping between SNP genotypes and phenotypic data using mixed linear models (MLM) to account for population structure.
QTL-seq v2.2.3 TASSEL 5.2.88 KASP assays (LGC Genomics) ChiPlot (clustering) Input: early bulk + late bulk BAM files Output: delta-SNP-index plot → QTL peaks → validated SNP markers

RNA Sequencing

RNA Sequencing Analysis

RNA reflects what a cell is actively doing. RNA-seq captures gene activity across conditions, time points, and even at the resolution of how individual genes are spliced. Choose your analysis pathway below.

1
RNA Extraction / Dataset Download
RNeasy kit · NCBI SRA · FASTQ files
+
What is RNA extraction? RNA is extracted from fresh tissue using kits like RNeasy (Qiagen) or TRIzol. Unlike DNA, RNA degrades very rapidly — samples must be snap-frozen in liquid nitrogen immediately after collection and kept at -80°C until processing.

RNA integrity: Before sequencing, RNA quality is assessed using a BioAnalyzer (RIN score — you want above 7) and quantified with Qubit. Good quality RNA is essential — degraded RNA produces unreliable expression data.

Library preparation: 1 µg of RNA is used per sample for library prep (e.g. Illumina TruSeq Stranded Total RNA with Ribo-Zero) to deplete ribosomal RNA and enrich for mRNA. Sequencing gives ~100 million paired-end reads per sample.

Public data: Same as DNA — NCBI SRA, prefetch + fasterq-dump.
RNeasy Kit (Qiagen) SRA Toolkit v3.4.1 Input: tissue / SRA accession Output: sample_1.fastq + sample_2.fastq 
prefetch SRR12345678
fasterq-dump SRR12345678.sra --split-files
2
Quality Check + Trimming
FastQC v0.12.1 · Trimmomatic v0.40 · Trim Galore! v0.6.7
+
Same principle as DNA — check quality with FastQC, trim adapters and low-quality bases with Trimmomatic, then run FastQC again to confirm improvement. For RNA-seq, the HEADCROP:10 parameter removes the first 10 bases to eliminate hexamer priming bias common in RNA library preparation.
FastQC v0.12.1 Trimmomatic v0.40 Input: raw .fastq files Output: trimmed paired .fastq files 
fastqc sample_1.fastq sample_2.fastq
trimmomatic PE sample_1.fastq sample_2.fastq \
  clean_1.fastq unpaired_1.fastq \
  clean_2.fastq unpaired_2.fastq \
  ILLUMINACLIP:TruSeq3-PE.fa:2:30:10 \
  HEADCROP:10 LEADING:3 TRAILING:3 \
  SLIDINGWINDOW:4:20 MINLEN:50
fastqc clean_1.fastq clean_2.fastq
3
Alignment — STAR / HISAT2 / TopHat
Splice-aware aligners · BAM output · IGV check
+
Why different aligners for RNA? RNA-seq reads come from mature mRNA, which has had introns removed by splicing. A read can span two exons that are thousands of bases apart on the genome. Regular aligners like BWA fail here — you need splice-aware aligners.

STAR: Spliced Transcripts Alignment to a Reference. The most widely used RNA-seq aligner. It builds a genome index and aligns reads in two passes — first discovering splice junctions, then aligning using those junctions. Very fast, handles novel splice sites. Used in our Arabidopsis splicing thesis.

HISAT2: Graph-based aligner, uses less memory than STAR, similar accuracy. Good alternative on memory-limited systems.

TopHat: Legacy splice-aware aligner (uses Bowtie2 internally). Used in the BCR-ABL CML project with the TopHat/Cufflinks pipeline.

After alignment: Load BAM in IGV to visually verify the alignment looks correct and to check strandedness of the data.
STAR v2.7.11b HISAT2 v2.2.1 TopHat (legacy) Input: clean .fastq + reference genome + GTF annotation Output: .BAM file sorted by coordinate 
# STAR alignment
STAR --runThreadN 8 \
  --genomeDir ./star_index/ \
  --readFilesIn clean_1.fastq clean_2.fastq \
  --outSAMtype BAM SortedByCoordinate \
  --outSAMattributes NH HI AS NM \
  --outFileNamePrefix ./output/sample_
samtools index ./output/sample_.Aligned.sortedByCoord.out.bam
4
Feature Counts — Count Matrix Generation
featureCounts v2.0.3 · HTSeq · Cufflinks (legacy)
+
What is a count matrix? After alignment, you count how many reads landed on each gene in each sample. The result is a matrix with genes as rows and samples as columns. Each cell is an integer — the number of reads from that sample that mapped to that gene. More reads = higher gene expression.

featureCounts is fast and memory-efficient, part of the Subread package. It assigns reads to genomic features based on a GTF annotation file.

Cufflinks (legacy): The older pipeline (used in the BCR-ABL project) assembled transcripts from BAM files, then used Cuffmerge, Cuffdiff, and CummeRbund for differential expression and visualisation.
featureCounts v2.0.3 HTSeq v2.0.2 Cufflinks / Cuffdiff (legacy) Input: .BAM files + annotation.gtf Output: count matrix (Gene x Sample TSV) 
featureCounts -T 8 -p \
  -a annotation.gtf \
  -o counts_matrix.txt \
  sample1.bam sample2.bam sample3.bam
5
Differential Expression — DESeq2 / edgeR / CopyKat
R v4.2.1 · statistical testing · volcano plots · expression tables
+
What is DESeq2? An R package that takes raw count data and determines which genes are significantly differentially expressed between conditions. It normalises for library size differences using median-of-ratios normalisation, applies a negative binomial statistical model, and uses shrinkage estimation for accurate fold-change estimates even with small sample sizes.

Significance threshold: A gene is considered differentially expressed if adjusted p-value (FDR) < 0.05 and absolute log2 fold change >= 1 (meaning 2x difference or more).

Visualisation: Volcano plots show all genes as dots — x-axis is fold change, y-axis is significance. Genes in the upper right are significantly upregulated; upper left are significantly downregulated. Gene expression plots and heatmaps show patterns across samples.
DESeq2 v1.38 (R) edgeR v3 (R) EnhancedVolcano (R) Input: count matrix Output: DGE results table (log2FC + padj per gene) + plots Significant: padj < 0.05 AND |log2FC| >= 1 
library(DESeq2)
dds <- DESeqDataSetFromMatrix(
  countData=counts, colData=coldata,
  design=~condition)
dds <- DESeq(dds)
res <- results(dds)
write.csv(as.data.frame(res),"DGE_results.csv")
6
Gene Expression Analysis — Plots and Tables
ggplot2 · heatmaps · PCA · expression profiling
+
After DESeq2, results are explored through multiple visualisations. PCA plots show whether samples cluster as expected by condition. Heatmaps show expression patterns across all significant genes. Box plots compare normalised expression of individual genes of interest. These visualisations are used in publications to communicate findings clearly.
ggplot2 (R) pheatmap (R) PCAtools (R) Output: publication-quality figures
7
Gene Ontology Enrichment and Pathway Analysis
clusterProfiler · gProfiler · AgriGO · RiceFREND · KEGG
+
Once you have your significant gene list, enrichment analysis maps them to biological functions and pathways. Gene Ontology classifies genes by biological process, molecular function, and cellular component. Enriched GO terms tell you what biological machinery is being affected. KEGG maps genes to metabolic pathways with visual diagrams. In our rice DGE study, DEGs were enriched in heading-date and flowering-time pathways — validating our experimental design.
clusterProfiler (R) gProfiler AgriGO RiceFREND Input: significant DEG list Output: enriched GO terms + KEGG pathways
1
Trimming + Quality Control
Trimmomatic v0.40 · FastQC again to confirm
+
Same trimming pipeline as DGE. For the Arabidopsis splicing thesis, Trimmomatic v0.40 was used with HEADCROP:10 (hexamer bias removal), SLIDINGWINDOW:4:20, and MINLEN:50. All 32 samples were trimmed with identical parameters before STAR alignment.
Trimmomatic v0.40 FastQC v0.12.1 Input: raw .fastq (32 samples) Output: 32 x trimmed paired .fastq
2
Alignment — STAR (splice-aware) + IGV strandedness check
STAR v2.7.11b · HISAT2 · --outSAMattributes required for SpliSER
+
For splicing analysis, STAR must be run with specific flags so SpliSER can work correctly. The --outSAMattributes NH HI AS NM tags are required by SpliSER to identify uniquely mapping reads. After alignment, strandedness is visually verified in IGV before proceeding — this determines whether to use the -s RF or -s FR flag in SpliSER.
STAR v2.7.11b IGV (strandedness check) Input: trimmed .fastq + TAIR10.1 genome Output: .BAM + .BAI per sample (32 total) 
STAR --runThreadN 8 \
  --genomeDir ${GENOME_DIR} \
  --readFilesIn clean_1.fastq clean_2.fastq \
  --outSAMtype BAM SortedByCoordinate \
  --outSAMattributes NH HI AS NM \
  --genomeSAindexNbases 12 \
  --outFileNamePrefix ${OUTPUT_DIR}/${sample_id}.
samtools index ${OUTPUT_DIR}/${sample_id}.Aligned.sortedByCoord.out.bam
3
SpliSER Process
SpliSER v1.0.4 · Balasubramanian Lab · alpha and beta read counting
+
What is SpliSER? SpliSER (Splice Site Efficiency from RNA-seq) counts two types of reads at every splice site: alpha reads (reads that cross the splice junction — they skipped the intron) and beta reads (reads that end at the splice site without crossing). The ratio gives a Splice Site Efficiency (SSE) score between 0 and 1 for each splice site in each sample.

SpliSER process is run independently on each of the 32 BAM files.
SpliSER v1.0.4 Input: sample.bam + genomic_clean.gtf Output: per-sample splice site counts (.tsv) 
spliser process sample.bam \
  -o spliser_output_sample \
  -A genomic_clean.gtf
# For stranded data: add -s RF or -s FR
4
SpliSER Combine + Output
Merges 32 samples · SSE normalisation · diffSpliSER format
+
SpliSER combine merges all 32 per-sample outputs into one unified dataset, interpolating missing counts for splice sites absent in any sample. SpliSER output then converts raw counts to normalised SSE values and formats the data for diffSpliSER statistical testing using the -t flag.
Input: 32 x SpliSER process outputs + samples_file.tsv Output: final_spliser_output.tsv (SSE matrix) 
spliser combine -s samples_file.tsv \
  -o spliser_combined_all_samples

spliser output -C spliser_combined_all_samples \
  -o final_spliser_output.tsv \
  -t diffspliSER
5
diffSpliSER — Pairwise Comparisons
edgeR v3 · 120 pairwise comparisons · FDR < 0.05 · |dSSE| >= 0.1
+
diffSpliSER uses edgeR's generalised linear model (GLM) framework to statistically test differential splice site usage between every pair of time point groups. With 16 groups (8 LLCT + 8 LDZT time points), this produces 120 pairwise comparisons. The contrast for each comparison is: (alpha.A - beta.A) - (alpha.B - beta.B), which tests whether the splice site efficiency differs between conditions. Significant sites must pass FDR < 0.05 AND |delta-SSE| >= 0.1.
edgeR v3 (R) diffSpliSER R Markdown Input: SSE matrix + target file (sample groups + library sizes) Output: 120 x pairwise results files (_hits.tsv)
6
Venn Diagram + PCA Analysis
VennDiagram v1.7.3 · ggplot2 · reproducibility across comparisons
+
Four-way Venn diagrams compare Dawn vs Night comparisons across LLCT and LDZT conditions to find splice sites that are reproducibly differentially spliced across all four comparisons. The four-way overlap represents the most reliable, biologically meaningful sites. PCA on SSE values confirms sample clustering by time point and condition.
VennDiagram v1.7.3 (R) ggplot2 + prcomp (R) Output: Venn diagrams + PCA plots
7
Motif Enrichment Analysis — XSTREME
MEME Suite · bedtools getfasta · CisBP-RNA Arabidopsis
+
600bp windows centred on each differentially spliced site are extracted from the TAIR10.1 genome using bedtools getfasta. XSTREME searches these sequences against the Arabidopsis CisBP-RNA RNA-binding protein motif database to identify which RBPs may be regulating splicing at these sites under circadian control. Randomly sampled control sequences (2000 or 5000 sites) serve as the background.
XSTREME (MEME Suite) bedtools getfasta CisBP-RNA database Input: splice site coordinates → 600bp FASTA sequences Output: enriched RBP motifs at circadian splice sites 
bedtools getfasta -fi genome.fa \
  -bed test_sites_600bp.bed \
  -fo test_sequences.fa -s -name

xstreme --oc output/ --dna --dna2rna \
  --evt 0.05 --minw 5 --maxw 10 \
  --align center --meme-mod zoops \
  --m Ray2013_rbp_Arabidopsis_thaliana.meme \
  --p test_sequences.fa \
  -n control_2000_sequences.fa
